# Root Orchestrator: Rules of Engagement
**Role:** Project Manager & Lead Architect
**Scope:** Root `/`, `/contracts`, `/specs`

## System Architecture
This project is a high-performance PostgreSQL Database Activity Monitoring (DAM) system. It uses a monorepo structure with a contract-first approach.

## Rules for Agents
1. **Contract-First:** All cross-component communication must be defined in `/contracts` (Protobuf/JSON Schema) before implementation.
2. **Monorepo Integrity:** Do not introduce duplicate dependencies across sub-packages.
3. **Model Context Protocol (MCP):** Use the shared MCP servers for Kubernetes and Git operations.
4. **Performance Budgets:** Every pull request must include a performance impact summary generated by the Performance Agent.
